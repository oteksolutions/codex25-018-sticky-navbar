About This Project

This sticky navbar implementation uses pure CSS without any JavaScript dependencies. I wanted to create a lightweight, accessible navigation solution that works across all modern browsers and devices.

The biggest challenge was creating a responsive mobile menu that doesn't require JavaScript. I solved this using the CSS "checkbox hack" which lets us toggle the mobile menu state using only HTML and CSS. This approach keeps the page lightweight while maintaining full functionality.

For the sticky behavior, I'm using the CSS position: sticky property which has excellent browser support these days. This provides smooth scrolling behavior without the jumpiness that can occur with JavaScript-based sticky implementations.

Performance was another key consideration. By keeping the navbar slim and using minimal CSS transitions, the scrolling remains smooth even on lower-powered mobile devices. The total size of the CSS is under 5KB when minified, making it lightning fast to load.

Features & Implementation

Browser Compatibility

Tested and working in Chrome, Firefox, Safari, and Edge. For older browsers that don't support position: sticky, I've included a simple fallback that defaults to a static position.

Accessibility

The navbar is fully keyboard navigable and includes appropriate ARIA attributes. The mobile menu toggle is properly labeled for screen readers, and all interactive elements have appropriate focus states.

Customization

The CSS variables in the root selector make it easy to customize colors, transitions, and sizing without digging through the CSS. You can quickly adapt this navbar to match your brand's color scheme.

CSS-Only Animation

The subtle hover effects and mobile menu transitions are all achieved using CSS, keeping the implementation lightweight while still providing visual feedback to users.

Frequently Asked Questions

Does this navbar work in Internet Explorer?

This navbar implementation uses position: sticky, which is not supported in IE11 and below. However, I've included a fallback that makes the navbar behave as a normal static element in unsupported browsers. If IE11 support is critical for your project, I recommend using a JavaScript-based sticky solution instead.

How do I change the navbar height?

The navbar height can be adjusted by modifying the padding in the #sticky-header selector. For the mobile version, you'll also want to adjust the corresponding values in the media query. If you're using the CSS variables version, simply modify the --nav-height variable in the :root selector.

Can I add dropdown menus to this navbar?

Yes! The current implementation doesn't include dropdowns, but they can be added with minimal additional CSS. I have a separate blog post about implementing accessible CSS-only dropdown menus that would integrate perfectly with this navbar system. The key is to use the :focus-within pseudo-class along with appropriate hover states.

Why not use JavaScript for the mobile menu?

While JavaScript would offer more control and features, this implementation prioritizes performance and reliability. The CSS-only approach means one less dependency, faster load times, and no risk of JavaScript errors breaking your navigation. For most projects, this simple approach is sufficient.

Is there a SASS/SCSS version available?

Yes! You can find the SCSS source files in the GitHub repository. The SCSS version includes variables, mixins, and nested selectors that make the code more maintainable for larger projects. It's located in the /scss directory of the repository.
